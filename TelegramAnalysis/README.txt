📋 ОПИС ПРОЕКТУ
---------------
Автоматизована система для аналізу російськомовних Telegram чатів з:
  • Збором даних через Telethon API
  • Категоризацією та валідацією через ChatGPT
  • Персистентним станом у SQLite
  • Rate limiting 150 чатів/годину
  • Батчовою обробкою по 10 активних чатів
  • Підтримкою множинних форматів вхідних даних

===================================================================================
🆕 НОВА АРХІТЕКТУРА (v2.0) - ЩО ЗМІНИЛОСЬ
===================================================================================

🔑 КЛЮЧОВІ ПОЛІПШЕННЯ:

  1. SQLite замість JSON
     • ACID гарантії для персистентності
     • Атомарні операції без race conditions
     • Автоматичні backups кожні 100 чатів

  2. In-Memory Cache
     • O(1) lookup для дедуплікації
     • Синхронізація з SQLite при старті
     • Економія ~99% перевірок файлової системи

  3. Rate Limiting
     • 150 чатів/годину з фіксованими вікнами
     • Персистентний стан між перезапусками
     • Автоматичне очікування наступного вікна

  4. Батчова Обробка
     • Збір 10 активних чатів з Telegram
     • Послідовна обробка через ChatGPT
     • Increment rate limit ТІЛЬКИ після успіху

  5. Множинні Формати Вводу
     • TSV: ID\tНазва\t@username (пріоритет 1)
     • JSON: {"chats": [...]} (пріоритет 2)
     • Text: Список t.me URLs (пріоритет 3)
     • Автовизначення формату

  6. Dual-Mode Input
     • Нова папка "чати/" для організованих даних
     • Legacy підтримка @mt_offer.txt
     • Обидва джерела працюють одночасно

⚙️ ВСТАНОВЛЕННЯ ТА НАЛАШТУВАННЯ
===================================================================================

1. ВИМОГИ
   ---------
   • Python 3.8+
   • Chrome/Edge браузер
   • Telegram API креди (api_id, api_hash)
   • Telethon session файл в accs/
   • ChatGPT Plus акаунт

2. ВСТАНОВЛЕННЯ ЗАЛЕЖНОСТЕЙ
   --------------------------
   pip install telethon playwright python-dotenv

3. НАЛАШТУВАННЯ .env
   ------------------
   Створіть файл .env з:

   API_ID=ваш_api_id
   API_HASH=ваш_api_hash

4. НАЛАШТУВАННЯ CHROME
   --------------------
   Запускається автоматично через RUN.bat або вручну:

   chrome.exe --remote-debugging-port=9222 --user-data-dir="C:\chrome-playwright-profile"

===================================================================================
▶️ ВИКОРИСТАННЯ
===================================================================================

1. ПІДГОТОВКА ДАНИХ
   -----------------

   Варіант A: Нова система (папка "чати/")
   ----------------------------------------
   Додайте файли в папку "чати/" у будь-якому форматі:

   • TSV формат (рекомендовано):
     123456	Crypto Чат	@crypto_chat
     789012	SMM Група	@smm_group

   • JSON формат:
     {
       "chats": [
         {"url": "https://t.me/example", "username": "example", "title": "Title"}
       ]
     }

   • Текстовий список URLs:
     https://t.me/chat1
     https://t.me/chat2

   Варіант B: Legacy система
   --------------------------
   Використовуйте @mt_offer.txt у форматі:
   ID	Назва	@username

2. ЗАПУСК СИСТЕМИ
   ---------------

   Звичайний запуск (продовжує з попереднього стану):

     python analyze_chats_playwright.py --auto

   Або подвійний клік на RUN.bat

   Запуск з очищенням попередніх результатів:

     python analyze_chats_playwright.py --auto --fresh-start

3. МОНІТОРИНГ ПРОЦЕСУ
   --------------------

   Загальна статистика:
     python utils/inspect_state.py

   Детальна інформація:
     python utils/inspect_state.py --detailed

   Статус rate limiter:
     python utils/inspect_state.py --rate-limit

   Останні 20 оброблених чатів:
     python utils/inspect_state.py --recent 20

   Останні 30 відхилених чатів:
     python utils/inspect_state.py --failed 30

4. ПЕРЕГЛЯД ЛОГІВ
   ---------------

   Логи зберігаються в: state/chats_spider.log

   Рівні логування:
     • INFO - основні події
     • WARNING - попередження
     • ERROR - помилки
     • DEBUG - детальна інформація

  │   ├─ Створення промпту
  │   ├─ Відправка в ChatGPT

  │   ├─ Збереження в output файли
  │   ├─ Mark processed в SQLite
  │   └─ Increment rate limiter (ТІЛЬКИ після успіху!)
  ├─ Очистка батчу
  ├─ Backup кожні 100 чатів
  └─ Повторити

📊 БАЗА ДАНИХ SQLite (state/chats.db)
===================================================================================

ТАБЛИЦЯ: processed_chats
  • url (PRIMARY KEY) - URL чату
  • processed_at - Час обробки
  • session_id - ID сесії
  • validation_result - "PASS" або "FAIL"
  • category - Категорія чату
  • owner - "founder" або "manager"
  • rejection_reason - Причина відхилення

ТАБЛИЦЯ: rate_limit
  • hour_window (PRIMARY KEY) - Годинне вікно (ISO format)
  • chats_processed - Кількість оброблених
  • last_update - Час останнього оновлення

ТАБЛИЦЯ: pending_batch
  • id (PRIMARY KEY) - ID запису
  • batch_number - Номер батчу
  • chat_url - URL чату
  • chat_data - JSON дані чату
  • added_at - Час додавання

🔧 TROUBLESHOOTING
===================================================================================

ПРОБЛЕМА: "База даних заблокована"
РІШЕННЯ: Закрийте всі інстанси скрипта, видаліть state/chats.db-journal

ПРОБЛЕМА: "Не знайдено session файлів"
РІШЕННЯ: Додайте .session файл в папку accs/

ПРОБЛЕМА: "Не вдалося підключитись до браузера"
РІШЕННЯ: Запустіть Chrome через RUN.bat або start_chrome.bat

ПРОБЛЕМА: "FloodWait від Telegram"
РІШЕННЯ: Система автоматично чекає. Або зменште DELAY_BETWEEN_CHATS

ПРОБЛЕМА: "Досягнуто ліміт 150 чатів/годину"
РІШЕННЯ: Система автоматично чекає наступного вікна. Це нормально.

ПРОБЛЕМА: "Всі чати вже оброблені"
РІШЕННЯ: Додайте нові чати або використайте --fresh-start для перезапуску

💡 ПОРАДИ ТА BEST PRACTICES
===================================================================================

1. ОРГАНІЗАЦІЯ ДАНИХ
   • Використовуйте підпапки в "чати/" для категорій
   • TSV формат - найнадійніший та найшвидший
   • Перевіряйте приклади в example_*.txt файлах

2. ПРОДУКТИВНІСТЬ
   • Система обробляє ~150 чатів/годину
   • 1000 чатів = ~6-7 годин роботи
   • Backups створюються автоматично кожні 100 чатів

3. МОНІТОРИНГ
   • Регулярно перевіряйте state/chats_spider.log
   • Використовуйте inspect_state.py для статистики
   • Перевіряйте pass rate (має бути >50%)

4. БЕЗПЕКА
   • Ніколи не діліться .session файлами
   • Не коммітьте .env в git
   • Регулярно робіть backup state/chats.db

5. ОПТИМІЗАЦІЯ
   • Видаляйте старі backups (старші 1 місяця)
   • Очищайте логи раз на тиждень
   • Використовуйте --fresh-start тільки при необхідності
