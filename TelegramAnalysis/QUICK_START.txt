📋 ПІДГОТОВКА (ПЕРШИЙ ЗАПУСК)
==================================================================

1. Перевірте що встановлено:
   □ Python 3.8+
   □ Telegram session файл в accs/
   □ .env файл з API_ID та API_HASH
   □ ChatGPT Plus акаунт

2. Підготуйте дані:

   Варіант A: Нова папка "чати/" (рекомендовано)
   -----------------------------------------------
   Створіть файл у папці "чати/" у будь-якому форматі:

   TSV (найкраще):
   123456	Crypto Talks	@crypto_talks
   789012	SMM Channel	@smm_channel

   JSON:
   {"chats": [{"url": "https://t.me/example"}]}

   Text URLs:
   https://t.me/chat1
   https://t.me/chat2

   Варіант B: Legacy @mt_offer.txt
   --------------------------------
   Формат: ID\tНазва\t@username

==================================================================
▶️ ЗАПУСК СИСТЕМИ
==================================================================

МЕТОД 1: ПОВНІСТЮ АВТОМАТИЧНИЙ (рекомендовано)
-----------------------------------------------
Подвійний клік на START_ALL.bat

⚡ Що робить:
   ✓ Перевіряє/створює віртуальне середовище
   ✓ Встановлює залежності якщо потрібно
   ✓ Запускає Chrome автоматично
   ✓ Запускає аналіз без підтверджень
   ✓ Обробляє всі чати безперервно

💡 Просто авторизуйтесь в ChatGPT у Chrome - все інше автоматично!

МЕТОД 2: Через командний рядок
-------------------------------
venv\Scripts\python.exe analyze_chats_playwright.py --auto

МЕТОД 3: З очищенням попередніх результатів
--------------------------------------------
venv\Scripts\python.exe analyze_chats_playwright.py --auto --fresh-start

==================================================================
📊 МОНІТОРИНГ ПРОЦЕСУ
==================================================================

Перевірка загальної статистики:
--------------------------------
python utils/inspect_state.py

Детальна інформація:
--------------------
python utils/inspect_state.py --detailed

Статус rate limiter:
--------------------
python utils/inspect_state.py --rate-limit

Останні оброблені чати:
-----------------------
python utils/inspect_state.py --recent 20

Перегляд логів:
---------------
tail -f state/chats_spider.log        # Linux/Mac
type state\chats_spider.log            # Windows

Знайдете результати в папці output/:

  validated_chats.txt    - Всі валідні чати (PASS)

⚡ ЩО РОБИТИ ЯКЩО...
==================================================================

❌ "Не знайдено session файлів"
→ Додайте .session файл в папку accs/

❌ "Не вдалося підключитись до браузера"
→ Запустіть RUN.bat (Chrome запуститься автоматично)
→ Авторизуйтесь в ChatGPT

❌ "Досягнуто ліміт 150 чатів/годину"
→ Це нормально! Система автоматично чекає наступної години

❌ "FloodWait від Telegram"
→ Система автоматично чекає. Це захист Telegram API

❌ "Всі чати вже оброблені"
→ Додайте нові чати в папку "чати/"
→ АБО запустіть з --fresh-start для повного перезапуску

==================================================================
💡 КОРИСНІ ПОРАДИ
==================================================================

1. Система обробляє ~150 чатів/годину
   → 1000 чатів = ~6-7 годин роботи

2. Backups створюються автоматично кожні 100 чатів
   → Знаходяться в state/chats_backup_*.db

3. Можна зупинити Ctrl+C - прогрес збережеться
   → Наступний запуск продовжить з того місця

4. Використовуйте TSV формат для найкращої швидкості
   → ID\tНазва\t@username

5. Організуйте дані в підпапки:
   чати/
     ├── крипто/
     ├── маркетинг/
     └── фріланс/
